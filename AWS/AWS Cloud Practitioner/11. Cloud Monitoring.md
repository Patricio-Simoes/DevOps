# 11. Cloud Monitoring

## CloudWatch

CloudWatch is a **monitoring & observability service that collects and analyzes metrics**, logs and events in real-time from AWS and on-prem resources.

It monitors resources like  [EC2](04.%20Amazon%20EC2.md#04.%20EC2), [Lambda ](07.%20Compute%20Services.md#lambda-functions-serverless), [EBS](04.%20Amazon%20EC2.md#ebs-volumes) and [RDS](06.%20Databases.md#amazon-rds) and can trigger alarms, dashboards and automated actions.

Integrates with auto scaling, [SNS](08.%20Integration%20Services.md#amazon-sns), [EventBridge](#eventbridge-serverless), etc.

### CloudWatch Logs

CloudWatch Logs is a **CloudWatch feature** for collecting, storing and analyzing **log data from AWS services**, custom applications and on-prem systems.

**Note:** Logs are stored in log groups and log streams.

## EventBridge (Serverless)

EventBridge is an event bus that routes events from AWS services, [SaaS](00.%20Cloud%20Concepts.md#software-as-sa-service-saas) or custom applications to targets like [Lambda ](07.%20Compute%20Services.md#lambda-functions-serverless), [SQS](08.%20Integration%20Services.md#amazon-sqs), [SNS](08.%20Integration%20Services.md#amazon-sns), etc.

## CloudTrail

CloudTrail is a service that **records API activity and user actions** across an AWS account.

It captures AWS console, SDK, CLI and API calls and stores logs in [S3](05.%20Amazon%20S3.md).

**Note:** Supports event history for the last 90 days for free.

## Health Dashboard

Health Dashboard is a personalized dashboard showing the status of AWS services and events impacting existing resources.

It notifies of outages, maintenance and security events and offers guidance on mitigation.

**Note:** Focuses only on services and regions relevant to the AWS account.
