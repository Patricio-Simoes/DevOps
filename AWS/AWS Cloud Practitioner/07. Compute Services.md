# 07. Compute Services

## Container Services

### Amazon ECS (Elastic Container Service)

Amazon ECS is a fully managed **container orchestration service** for running Docker containers in AWS.

**Note:** While ECS handles placement, scaling and scheduling of containers, these will run on user-managed [EC2](04.%20Amazon%20EC2.md#04--ec2) instances.

### AWS Fargate (Serverless)

AWS Fargate is a c**ompute engine for containers** that works with [ECS](#amazon-ecs-elastic-container-service) and [EKS](#amazon-eks-elastic-kubernetes-service).

<div align="center">
  <img src="../../Assets/Images/AWS - Fargate.png" alt="AWS - Fargate" />
</div>

**Note:** There is no need to manage [EC2](04.%20Amazon%20EC2.md#04--ec2) instances when using Fargate.

### Amazon ECR (Elastic Container Registry)

Amazon ECR is a fully managed, **private Docker image repository** for storing and managing container images.

It is integrated with [ECS](#amazon-ecs-elastic-container-service) and [Fargate](#aws-fargate-serverless).

<div align="center">
  <img src="../../Assets/Images/AWS - ECR.png" alt="AWS - ECR" />
</div>

**Note:** Supports lifecycle policies and [IAM](03.%20IAM.md)-based access control.

### Amazon EKS (Elastic Kubernetes Service)

Amazon EKS is a **managed [Kubernetes](01.%20Overview.md#01--overview) control plane** for running containerized applications using Kubernetes.

It supports standard `kubectl`commands and AWS managed the k8s control plane, while the user managed the worker nodes (via [EC2](04.%20Amazon%20EC2.md#04--ec2) or [Fargate](#aws-fargate-serverless)).

## Lambda Functions (Serverless)

Lambda is a service to run **event driven functions** without provisioning or managing servers.

Key points:

- Automatically scales with demand;
- Pay-per-request and duration billing;
- Supports triggers from [S3](05.%20Amazon%20S3.md), [Amazon API Gateway](08.%20Integration%20Services.md#amazon-api-gateway), EventBridge and more;
- Ideal for short-lived tasks (e.g. Image processing, data transformation).

## AWS Batch

AWS Batch is a fully managed service to **run batch computing jobs** efficiently at any scale.

Key points:

- Automatically provisions compute ([EC2](04.%20Amazon%20EC2.md#04--ec2));
- Jobs are defined using Docker images;
- Runs on [ECS](#amazon-ecs-elastic-container-service) in the background;
- Ideal for analytics, simulations or large scale data processing.
